    这是自动备份系统客户端。
    作用:
         1. 监控指定备份目录，当这个目录下有新文件到来且未被占用时，则将该文件上传至备份服务器
         2. 如果用户输入恢复文件命令，则备份服务器将用户所上传的文件发送给客户端

    功能模块:
        1. 连接服务器模块:
            与中转服务器、备份服务器分别建立TCP连接
        2. 监控目录模块:
            不断监控备份目录是否有新文件产生
        3. 文件上传模块: 
            将备份文件上传至中转服务器
        4. 备份文件恢复模块:
            将备份的文件从备份服务器上下载到本地客户端指定目录

    客户端有三个线程:
        1. 主线程 -> 登录中转服务器，与中转服务器和备份服务器建立TCP连接
        2. 目录监控线程 -> 每当文件上传线程将上传队列中的文件上传完毕之后，该线程开始监控客户端的备份目录，将未上传的文件添加到上传队列，并判断文件是否被占用，设置占用标志
        3. 文件上传线程 -> 将文件传输队列中的文件一个接一个上传至中转服务器, 当一个文件上传完毕之后，就将该文件删除
